HTML Tag Masterclass — Classroom Reference

How to use this page

This page groups HTML tags by category. Each tag includes a short explanation and a small example you can copy. Use the search box to find a tag quickly (e.g. type img or form).

Tip: search matches tag names and short descriptions. Click any "Copy" button to copy the example to your clipboard.

Document & Metadata

Tags that define the document, encoding, and resources in the <head>.

<!DOCTYPE html>

Declares the document type and enables standards mode in browsers.

<!DOCTYPE html>

<html lang="...">

Root element of the document. The lang attribute helps accessibility and search engines.

<html lang="en"> ... </html>

<head> & <title>

<head> contains metadata; <title> sets the page title shown in browser tabs.

<head>
  <title>My Portfolio</title>
</head>

<meta>

Provides metadata like charset, viewport, and description for SEO and proper rendering.

<meta charset="utf-8">
<meta name="viewport" content="width=device-width,initial-scale=1">
<meta name="description" content="Portfolio of Ejistan">

<link>

Links external resources (CSS, icons, fonts). Use rel and href.

<link rel="stylesheet" href="css/style.css">
<link rel="icon" href="/favicon.png">

<style> & <script>

Inline CSS (<style>) and inline scripts or references to external JS (<script>).

<style> /* inline CSS */ </style>
<script src="js/main.js" defer></script>

Structure & Layout

Semantic elements to organize content into meaningful regions.

<header>

Represents introductory content, often contains branding and navigation.

<header>
  <h1>Site Title</h1>
</header>

<nav>

Navigation landmark containing links to other pages or sections.

<nav aria-label="Primary">
  <ul><li><a href="#about">About</a></li></ul>
</nav>

<main>

Main content area — there should be only one per page.

<main id="main-content"> ... </main>

<section> & <article>

<section> groups related content; <article> represents a self-contained piece (e.g., blog post).

<section id="features">...</section>
<article><h2>Post Title</h2><p>Excerpt...</p></article>

<aside>

Sidebar or tangential content related to the main content (e.g., widgets.)

<aside class="sidebar">Newsletter widget</aside>

<footer>

Footer for site information, copyright, or secondary navigation.

<footer>&copy; 2025 Ejistan</footer>

<div>

Generic container used when no semantic element is suitable.

<div class="card">...</div>

Headings & Text Formatting

Tags that create headlines, paragraphs, and text emphasis.

<h1> ... <h6>

Heading elements define the document structure and are important for accessibility and SEO.

<h1>Main Title</h1>
<h2>Section title</h2>

<p> & <span>

<p> is for paragraphs. <span> is an inline wrapper for styling or scripts.

<p>This is a paragraph.</p>
<span class="muted">Inline note</span>

<strong>, <em>, <b>, <i>

Semantic emphasis: <strong> indicates importance; <em> adds emphasis. <b> and <i> are presentational.

<strong>Important</strong> — <em>emphasis</em>

<u>, <small>, <mark>

Underline, fine print, and highlighted text respectively.

<u>Underlined</u> — <small>small text</small> — <mark>highlight</mark>

<br> & <hr>

Line break and thematic break (horizontal rule) respectively.

Line one.<br>Line two.
<hr>

<blockquote> & <cite>

Quotes and citation sources.

<blockquote>This is a quote.<footer><cite>— Author</cite></footer></blockquote>

<abbr>

Abbreviations with an optional title attribute to explain the abbreviation to assistive tech.

<abbr title="HyperText Markup Language">HTML</abbr>

Lists & Links

How to create lists and link between pages or sections.

<ul>, <ol>, <li>

Unordered and ordered lists with list items.

<ul>
  <li>Item one</li>
  <li>Item two</li>
</ul>

<a href="...">

Anchor tag creates links. Use href for URLs or anchors like #contact. Include rel="noopener" for external links.

<a href="https://example.com" target="_blank" rel="noopener">Visit example</a>

<base>

Sets a base URL for relative links. Use carefully — affects all relative paths on the page.

<base href="https://example.com/">

Media & Visuals

Images, responsive images, videos, and embedded content.

<img>

Images must include descriptive alt text for accessibility. Use loading="lazy" to defer offscreen images.

<img src="images/product.jpg" alt="SmartBand product" width="600" height="400" loading="lazy">

<picture> & <source>

Serve responsive or modern formats (WebP) with <picture> and <source>.

<picture>
  <source type="image/webp" srcset="product-400.webp 400w, product-800.webp 800w">
  <img src="product-800.jpg" alt="Product">
</picture>

<figure> & <figcaption>

Group images with captions for semantic meaning.

<figure>
  <img src="demo.jpg" alt="Demo">
  <figcaption>Screenshot of the demo</figcaption>
</figure>

<video> & <source>

Embed videos with multiple sources and controls.

<video controls width="640">
  <source src="clip.webm" type="video/webm">
  <source src="clip.mp4" type="video/mp4">
</video>

<iframe>

Embed external content like maps or third-party widgets. Use sandboxing and title for accessibility.

<iframe src="https://www.youtube.com/embed/..." title="Demo video" width="560" height="315" allowfullscreen></iframe>

<canvas>

Canvas for drawing graphics via JavaScript (advanced use).

<canvas id="myCanvas" width="300" height="150">Fallback text for older browsers.</canvas>

Forms & Inputs

Collecting user input — contact forms, subscriptions, and accessible labels.

<form>

Wraps input controls and defines submission action and method.

<form action="/submit" method="post">...</form>

<label> & <input>

Always pair <label for="id"> with inputs — better accessibility.

<label for="email">Email</label>
<input id="email" type="email" name="email" required>

<textarea> & <select> & <option>

Multi-line input and select controls.

<textarea rows="4" name="message"></textarea>
<select name="plan"><option value="basic">Basic</option></select>

<button type="submit">

Buttons for submission or client-side actions.

<button type="submit">Send</button>

<fieldset> & <legend>

Group related controls and provide a legend for context (useful for accessibility).

<fieldset><legend>Contact</legend>...</fieldset>

Tables & Data

Display tabular or structured data; use for pricing or comparison tables.

<table> & <caption>

Table root and optional caption describing the table.

<table>
  <caption>Pricing Plans</caption>
  <thead><tr><th>Plan</th></tr></thead>
  <tbody><tr><td>Basic</td></tr></tbody>
</table>

<thead>, <tbody>, <tr>, <th>, <td>

Head, body, rows, header cells, and data cells respectively.

<thead><tr><th>Name</th></tr></thead>

Semantic & Accessibility

Tags and features that improve meaning and accessibility for assistive tech and search engines.

<time>

Represents a specific time or date. Include datetime attribute for machine readability.

<time datetime="2025-10-17">Oct 17, 2025</time>

<address>

Contact information for person or organization (not for arbitrary addresses in text).

<address>Ejistan Tech, Accra</address>

<details> & <summary>

Expandable content that works without JS — great for FAQ sections.

<details><summary>FAQ question</summary><p>Answer</p></details>

<progress> & <meter>

Visual indicators of progress or measurements; use semantic attributes for values.

<progress value="40" max="100"></progress>
<meter value="6" min="0" max="10">6/10</meter>

ARIA & roles (attribute examples)

ARIA attributes and roles provide assistive tech hints — use them where native semantics are missing.

<nav aria-label="Main navigation">...</nav>
<div role="button" tabindex="0">Custom button</div>

Scripting & Interactivity

Tags used for scripts, progressive enhancement, and advanced UI patterns.

<script>

Include JS inline or reference external files. Use defer to delay execution until after parsing.

<script src="js/main.js" defer></script>

<noscript>

Fallback content for users with JS disabled.

<noscript>Please enable JavaScript to use this site.</noscript>

<template> & <dialog>

Templates for cloning DOM fragments and dialog for modals (modern browsers).

<template id="card-template"><div class="card">...</div></template>
<dialog id="confirm">Confirm action</dialog>

Code Display & Classroom Controls

How we display tutorials: <pre> preserves formatting; <code> marks code; buttons add copy-to-clipboard behavior.

<pre> & <code>

Always wrap code examples in <pre><code>...</code></pre> to preserve whitespace and allow copy behavior.

<pre><code> <!-- example code --> </code></pre>

<button> (copy buttons)

Buttons trigger copy actions. Add labels and accessible names for screen readers.

<button aria-label="Copy code">Copy</button>

Miscellaneous & Head Utilities

Other useful tags related to linking and document root behavior.

<base>

Sets the base URL for relative links in the document.

<base href="https://ejistan.example/">

Favicon (via <link>)

Small icon displayed in the browser tab.

<link rel="icon" href="/favicon.png" type="image/png">
