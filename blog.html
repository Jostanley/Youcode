Product Landing Page (Responsive)

Ejistan Tech — Classroom Tutorial (copyable code on every block)

Overview & learning goals

This lesson walks students through building a professional product landing page for Ejistan Tech, a fictional brand that sells a smart gadget. The page demonstrates modern layout techniques, marketing-focused copy structure, accessible markup, responsive behavior, and lightweight interactivity. Students will learn:

  • How to plan and structure a landing page for conversion (headline, hero, benefits, social proof, pricing, CTA).
  • How to write semantic HTML that supports accessibility and SEO.
  • How to use CSS (Flexbox + Grid + media queries) to create responsive layouts.
  • How to build reusable components — hero, feature cards, pricing cards, testimonials.
  • Performance and optimization tips for production-ready pages.

How to use this file: Copy code blocks into your editor. There is a Copy button in each block for convenience. Replace placeholder images and copy with your own content to practice customization.

2 — Content & conversion flow

Landing pages are marketing-first. Before coding, sketch the content flow. A classic, high-converting flow is:

  1. Hero (big promise + primary CTA)
  2. Value propositions / features (3–4 items)
  3. Social proof (testimonials, logos)
  4. Pricing or packages
  5. Secondary CTAs and FAQ
  6. Footer & legal

Student task: Write a one-sentence value proposition for Ejistan Tech. Example: "Ejistan SmartBand — track your health effortlessly with hospital-grade accuracy."

12 — Minimal JS for small interactions

Keep JS focused and small: form UX, mobile nav toggle, price toggle (monthly/yearly). Below is a minimal script for disabling the submit button after submission and for a mobile menu toggle.

// js/main.js (minimal)
document.addEventListener('DOMContentLoaded', () => {
  // Prevent double submit
  const form = document.getElementById('contact-form');
  if (form) {
    form.addEventListener('submit', e => {
      const btn = form.querySelector('button[type="submit"]');
      if (btn) {
        btn.disabled = true;
        btn.textContent = 'Sending...';
      }
    });
  }

  // Mobile nav toggle (assume .nav-toggle exists)
  const toggle = document.querySelector('.nav-toggle');
  if (toggle) {
    toggle.addEventListener('click', () => {
      document.querySelector('.nav').classList.toggle('open');
    });
  }
});

Performance tip: Defer or load JS asynchronously to avoid blocking initial paint. Keep the total JS bundle small.

13 — Performance & production checklist

Before students push to production, go over this checklist:

  • Optimize images (WebP/AVIF, responsive sizes with srcset).
  • Preconnect to font hosts & use display=swap.
  • Minify CSS/JS; enable gzip/Brotli on the server.
  • Use a CDN and set long cache headers for static assets.
  • Run Lighthouse and fix high-impact issues (LCP, FID, CLS).

14 — SEO & structured data

Teach the basics of SEO: meta title, meta description, and JSON-LD structured data for Product. Example JSON-LD helps search engines understand product details.

<script type="application/ld+json">
{
  "@context":"https://schema.org/",
  "@type":"Product",
  "name":"Ejistan SmartBand 2.0",
  "image":["https://example.com/images/product-hero.png"],
  "description":"Medical-grade SmartBand with continuous heart rate and sleep tracking.",
  "brand":{"@type":"Brand","name":"Ejistan"},
  "offers":{
    "@type":"Offer",
    "price":"99.00",
    "priceCurrency":"USD",
    "availability":"https://schema.org/InStock"
  }
}
</script>

Classroom exercise: Have students modify the JSON-LD fields for a different currency or a sale price and observe how structured data appears in the Rich Results Test tool.

15 — Accessibility checklist

Accessibility is essential. Go through this checklist with students and test manually:

  • Use semantic HTML (<header>, <main>, <nav>, <footer>).
  • Provide alt text for images (meaningful or empty for decorative).
  • Ensure sufficient color contrast (WCAG AA).
  • Make interactive controls keyboard-accessible and visible on focus.
  • Use proper labels for forms and ARIA where necessary.

Use tools like Lighthouse, axe, and manual keyboard testing. Encourage students to tab through the page and use a screen reader if available.

16 — Classroom assignments & challenges

Assign these tasks to reinforce learning. Each task escalates in complexity and adds value to the student's portfolio:

  1. Build & style the page: Implement the HTML/CSS provided and replace placeholder images with real product shots.
  2. Responsive polish: Ensure hero, features, and pricing look great on phones and tablets. Test multiple breakpoints.
  3. Performance sprint: Optimize the hero image and reduce LCP under 2.5s on a mobile 3G throttled test.
  4. Accessibility lab: Fix all Lighthouse accessibility issues and record before/after screenshots.
  5. Deploy: Publish on Netlify or Vercel and share the live link in the class forum.

Optional bonus: Add a simple checkout simulation using client-side JavaScript (no real payments). This demonstrates form handling and UX flow.

19 — Deploying the landing page

Deploy to GitHub Pages, Netlify or Vercel — all are free for static pages. For many classes, Netlify is simple (drag & drop or connect to Git), and it supports serverless functions if students want to handle forms later.

# Quick Netlify deploy (class demo)
1. git init && git add . && git commit -m "initial"
2. push to GitHub (create repo)
3. login to Netlify -> New site -> Import from GitHub
4. choose repo, build settings: (no build for static), publish directory: /
5. deploy and enable https

Security & privacy: If you collect emails, ensure you have a privacy policy and follow local regulations (GDPR/CCPA). For simple class demos, use email links or Netlify Forms.

20 — Final project rubric (grading criteria)

Provide students with a rubric to self-assess or to be used for grading:

  • Structure & semantics (20%) — correct use of headings, sections, and landmarks.
  • Responsiveness (20%) — layouts adapt cleanly at multiple breakpoints.
  • Design & visual polish (20%) — spacing, typography, imagery, and CTA clarity.
  • Accessibility & SEO (20%) — alt text, focus states, meta tags, JSON-LD.
  • Performance & deployment (20%) — images optimized, hosted live, and LCP measured.

If students meet 80% or above, they demonstrate a strong working landing page suitable for their portfolio.
