Creating Multi-Page Websites

A student tutorial with copiable code examples — detailed & practical

Overview & Learning Goals

This lesson teaches how to build, organize, and deliver a practical multi-page website. You will learn:

  • How to plan a site and create a clean folder structure
  • How to create navigation bars and footers that appear on every page
  • How to link pages using relative and absolute paths
  • How to create reusable header/footer code patterns (templates and server-side includes)
  • Responsive navigation and accessible markup
  • Optimization, SEO basics, and deployment tips

Note: This page is intentionally thorough. Read each section and copy the examples into your editor to practice. Many blocks include a Copy button to make it easy.

1. Plan Before You Build

Good websites start with planning. Before writing code, decide the pages you need, the content on each, and how users will move through the site.

What to include in your plan

  1. Site Map: List pages (Home, About, Services, Blog, Contact).
  2. Navigation Flow: Which pages link to which; what primary & secondary navigation looks like.
  3. Content Structure: For each page decide headings, text, images, forms.
  4. Design System: Colors, typography, spacing, and component patterns (buttons, cards).
  5. Device Targets: Mobile-first or responsive? Plan for breakpoints.

Example site map: Home → Services → Pricing → Contact → Blog (list & article pages)

Quick checklist

  • Which pages are required?
  • Which pages will be linked from the nav?
  • Will you use a templating system or pure static files?

2. Folder Structure (Keep it tidy)

A clear folder structure makes a multi-page site manageable. Here is a simple and effective one:


/my-website/
├─ index.html
├─ about.html
├─ services.html
├─ contact.html
├─ blog/
│  ├─ index.html
│  ├─ article-1.html
│  └─ article-2.html
├─ css/
│  └─ styles.css
├─ js/
│  └─ scripts.js
├─ images/
│  └─ hero.jpg
└─ assets/
   └─ fonts/

Explanation: Keep each file in logical places. All CSS in /css/, JavaScript in /js/, and images in /images/. The blog gets its own folder because it contains many pages.

When linking between pages, the directory depth matters — use relative paths like ../ when linking from nested folders.

3. Basic HTML Template for Every Page

Create a base template that every page follows: a consistent <head>, a header with navigation, and a footer. This makes the site coherent and easy to maintain.





  
  
  Page Title - My Site
  


  <!-- Site header (navigation) -->
  <header>
    <nav>
      <a href="/index.html">Home</a>
      <a href="/about.html">About</a>
      <a href="/services.html">Services</a>
      <a href="/blog/index.html">Blog</a>
      <a href="/contact.html">Contact</a>
    </nav>
  </header>

  <main>
    <!-- Page content -->
  </main>

  <footer>© 2025 My Site</footer>

Explanation: Notice that the <link rel="stylesheet"> points to a shared stylesheet and the script to a shared JS file. Using the same template on every page ensures consistent fonts, colors, and behavior.

4. Navigation Basics — Structure & Accessibility

Navigation is one of the most important UI elements. It should be easy to find, keyboard accessible, and semantic.

Semantic nav with accessibility features

Always use <nav> for primary navigation and include ARIA attributes where needed. Provide skip links for keyboard users to jump to main content.


<header>
  <a class="skiplink" href="#main-content">Skip to main content</a>
  <nav aria-label="Main navigation">
    <ul>
      <li><a href="/index.html">Home</a></li>
      <li><a href="/about.html">About</a></li>
      <li><a href="/services.html">Services</a></li>
      <li><a href="/blog/index.html">Blog</a></li>
      <li><a href="/contact.html">Contact</a></li>
    </ul>
  </nav>
</header>

Explanation: The skip link helps screen reader and keyboard users skip repetitive content. The aria-label describes the nav role to assistive tech.

CSS for a simple horizontal nav


/* css/styles.css */
header nav ul {
  list-style:none;
  display:flex;
  gap:18px;
  padding:0;
  margin:0;
}
header nav a {
  text-decoration:none;
  color: #222;
  padding:8px 10px;
  border-radius:6px;
}
header nav a:hover,
header nav a:focus {
  background:#e9f2ff;
}
.skip-link { position:absolute;left:-9999px;top:auto;width:1px;height:1px;overflow:hidden }
.skip-link:focus { position:static;width:auto;height:auto; }

Explanation: The nav uses flexbox to align links horizontally. The .skip-link is visually hidden but becomes visible when focused.

5. Linking Pages — Relative vs Absolute Paths

Understanding link paths is crucial in multi-page sites. There are two main types:

  • Absolute paths: Start with a slash / and point from the domain root: /about.html. Use these when files are served from the site root.
  • Relative paths: Based on the current file’s location: ../index.html or contact.html. Use when linking within subfolders.

Examples


/* From /blog/article-1.html to home (absolute) */
<a href="/index.html">Home</a>

/* From /blog/article-1.html up one folder to blog index (relative) */
<a href="../index.html">Blog home</a>

/* From /about.html to contact (relative, same folder) */
<a href="contact.html">Contact</a>

If you deploy to a subfolder (e.g. GitHub Pages with username.github.io/repo), absolute root paths may break. In that case use relative paths or set a base URL.

Using <base> (be careful)


<!-- Put in the head if you want to set a base URL for all relative links -->
<base href="/my-subfolder/">

Tip: For beginners, using consistent absolute paths starting with / is often simpler when deploying to root. If not using root, prefer relative links.

6. Reusable Header & Footer — Techniques

Maintaining identical headers and footers across many pages is tedious if edited manually. Use one of these patterns to keep code DRY (Don't Repeat Yourself).

Option A — Server-side includes (recommended on servers)

If you have a server that supports includes (PHP, Node/Express, or Apache SSI), create partials.


/* Example with PHP (header.php & footer.php) */

/* header.php */
<header>
  <nav>...links...</nav>
</header>

/* footer.php */
<footer>© 2025 My Site</footer>

/* Any page e.g. index.php */
<?php include 'header.php'; ?>
<main>...</main>
<?php include 'footer.php'; ?>

Explanation: This method ensures updates to header.php affect all pages. Use PHP, templating engines (e.g., EJS, Pug), or server frameworks the same way.

Option B — Client-side include (JavaScript)

If you're using static hosting without server-side templates, you can insert header/footer via JS. Note: This adds a tiny flash as content loads.


/* header.html (partial file) */
<nav>
 <ul>
   <li><a href="/index.html">Home</a></li>
   <li><a href="/about.html">About</a></li>
 </ul>
</nav>>

/* In your page's body where you want header to appear */
<div id="site-header"></div>

/* js/scripts.js (run before DOMContentLoaded or at end of body) */
function includeHTML(selector, url) {
  fetch(url).then(res => res.text()).then(html => {
    document.querySelector(selector).innerHTML = html;
  }).catch(err => console.error('Include failed', err));
}
includeHTML('#site-header','/partials/header.html');

Tradeoffs: Server-side templates are preferred for SEO and performance. Client-side includes work for static hosts, but search engines may see less content unless they render JS.

7. Responsive Navigation (Mobile-friendly)

On small screens, the main nav should collapse into a menu toggle (hamburger). This example shows a simple accessible pattern using CSS and minimal JS.

HTML


<header class="site-header">
  <button id="nav-toggle" aria-expanded="false" aria-controls="primary-nav">
    <span class="sr-only">Open menu</span>
    ☰
  </button>

  <nav id="primary-nav" aria-label="Primary" class="nav-collapsed">
    <ul>
      <li><a href="/index.html">Home</a></li>
      <li><a href="/services.html">Services</a></li>
      <li><a href="/contact.html">Contact</a></li>
    </ul>
  </nav>
</header>

CSS


/* css/styles.css (add) */
.site-header{display:flex;justify-content:space-between;align-items:center;padding:12px}
.nav-collapsed ul{display:flex;gap:12px;margin:0;padding:0;list-style:none}
@media (max-width:700px){
  .nav-collapsed ul{display:block;max-height:0;overflow:hidden;transition:max-height .25s ease}
  .nav-collapsed.open ul{max-height:400px} /* expanded */
  #nav-toggle{display:inline-block}
}

JS (toggle)


/* js/scripts.js */
document.getElementById('nav-toggle')?.addEventListener('click', function(){
  const nav = document.getElementById('primary-nav');
  const open = nav.classList.toggle('open');
  this.setAttribute('aria-expanded', open ? 'true' : 'false');
});

Accessibility tips: Ensure the toggle button updates aria-expanded. Provide visible focus outlines for keyboard users and avoid hiding links for screen readers.

8. Building Professional Footers

Footers usually include contact links, legal links, site map, and small print. Keep them structured semantically inside a <footer> element.

Footer example


<footer class="site-footer">
  <div class="footer-columns">
    <div>
      <h4>About</h4>
      <p>Short site description.</p>
    </div>
    <div>
      <h4>Links</h4>
      <ul>
        <li><a href="/privacy.html">Privacy</a></li>
        <li><a href="/terms.html">Terms</a></li>
      </ul>
    </div>
    <div>
      <h4>Contact</h4>
      <address>Email: <a href="mailto:hello@example.com">hello@example.com</a></address>
    </div>
  </div>
  <p style="text-align:center;margin-top:10px">&copy; 2025 My Site</p>
</footer>

Design: Use a subdued background, smaller text, and well-spaced columns. Keep legal links accessible in the footer.

9. Breadcrumbs & Page Hierarchy

Breadcrumbs help users understand where they are within a site. They are also helpful for SEO.


<nav aria-label="Breadcrumb">
  <ol class="breadcrumb">
    <li><a href="/blog/index.html">Blog</a></li>
    <li>How to build navs</li>
  </ol>
</nav>

Tip: Use ordered lists for breadcrumbs and separate style vs semantics — this helps assistive tech.

11. Contact Forms across Pages

Forms are common across pages. Use consistent validation UX and server endpoints. Here’s a basic accessible contact form with HTML5 validation.


<form action="/api/contact" method="post" novalidate>
  <label for="name">Name</label>
  <input id="name" name="name" required>

  <label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <label for="message">Message</label>
  <textarea id="message" name="message" rows="5" required></textarea>

  <button type="submit">Send</button>
</form>

Server note: Make sure your server endpoint sanitizes input and returns proper 200/400 responses. On static hosting you can use services like Formspree or Netlify Forms.

12. SEO Basics for Multi-Page Sites

SEO helps your pages get found. For multi-page sites, be consistent and provide metadata on each page.

Essential meta tags


<!-- in the head of each page -->
<title>Services - My Site</title>
<meta name="description" content="Professional services to help you... ">
<link rel="canonical" href="https://example.com/services.html" />

Other tips: Use structured data (JSON-LD) for articles and businesses, create an XML sitemap and a robots.txt, and make sure each page has unique content and title tags.

13. Performance & Best Practices

Faster pages improve UX & SEO. Important steps:

  • Compress and serve optimized images (WebP when possible)
  • Minify CSS and JS
  • Use browser caching and a CDN for assets
  • Load non-essential scripts asynchronously

Lazy load images


<img src="hero.jpg" alt="Hero" loading="lazy" width="1200" height="600">

Using loading="lazy" defers off-screen images and speeds up initial paint.

14. Fonts & Icons

Use web fonts and icons carefully. Keep a fallback font stack for performance and accessibility.

Google Fonts example


<!-- head -->
<link rel="preconnect" href="https://fonts.gstatic.com">
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap" rel="stylesheet">

/* css */
body{font-family:'Inter',system-ui,-apple-system,Segoe UI,Roboto,Arial,sans-serif}

Icons: Use an icon font or SVG sprite. SVGs are scalable and accessible (add role="img" and <title> for accessibility).

15. Analytics, Privacy & Legal

Tracking user behavior is useful but must respect privacy. Provide a clear privacy policy and consider adding cookie consent if you track users.

Simple Google Analytics snippet (GA4)


<!-- head -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXX');
</script>

Replace G-XXXXXXX with your measurement ID. For privacy sensitive sites, consider cookieless analytics or server-side tracking.

16. Testing & Deployment

Before deploying, test on multiple devices and run Lighthouse to check performance and accessibility. When ready, deploy to a static host or a server.

Deployment options

  • Static hosting: GitHub Pages, Netlify, Vercel (good for static sites)
  • Server hosting: DigitalOcean, AWS, or shared hosting (when you need backend)
  • CDN: Use Cloudflare for additional caching and security

Quick deploy with Netlify (example)


/* Steps:
1. Create a Git repo of your site.
2. Sign in to Netlify -> New site -> Import from Git.
3. Select repo and deploy.
4. Set publish directory to root or /dist if using a build step.
*/

Tip: Use HTTPS and an SSL certificate (most hosts provide one automatically).

18. Common Issues & Troubleshooting

Broken links

Check your paths carefully. Use browser devtools to see 404s. If a link points to /css/styles.css but your site is deployed under /repo/, it may need /repo/css/styles.css or a relative path.

Resources not loading

Check console errors for CORS, wrong MIME types, or blocked mixed content (HTTP vs HTTPS).

SEO issues

Make sure each page has a unique <title> and meta description. Use server-side rendering for dynamic sites to ensure crawlers index content.
