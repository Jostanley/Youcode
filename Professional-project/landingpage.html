Product Landing Page (Responsive)

Overview & learning goals

In this project you'll build a responsive product landing page suitable for SaaS products, apps, or digital tools. The page will include:

  • A compelling hero section with primary CTA
  • Feature highlights using grid & cards
  • Pricing section with tiers and CTA
  • Testimonials and social proof
  • Accessible markup, responsive behavior (mobile-first), and performance tips

Follow step-by-step: copy examples into files (index.html & style.css), then customize branding, images, and text for your product.

1 — Plan & folder structure

Keep an organized project folder:

/product-landing/
  index.html
  css/
    style.css
  images/
    hero.png
    feature-1.jpg
    testimonial-1.jpg
  js/
    main.js
  assets/
    logos/

Why: Clear structure makes linking assets easy and deployment predictable (GitHub Pages, Netlify, Vercel).

2 — Mobile-first HTML skeleton

Start with semantic HTML and simple content placeholders. Keep navigation, hero, features, pricing, testimonials, and footer.

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Product Name — Landing Page</title>
  <link rel="stylesheet" href="css/style.css">
</head>
<body>

  <header>
    <nav class="site-nav" aria-label="Main Navigation">
      <div class="logo">ProductName</div>
      <div class="nav-actions">
        <a href="#" class="btn ghost">Sign in</a>
        <a href="#" class="btn">Get started</a>
      </div>
    </nav>
  </header>

  <main>
    <!-- Hero, features, pricing, testimonials go here -->
  </main>

  <footer> &copy; 2025 ProductName </footer>
  <script src="js/main.js"></script>
</body>
</html>

Accessibility: Include aria-label on navigation and ensure the page has one logical <main> landmark.

3 — Hero section: message, visual, CTA

The hero is the most important real estate. Keep a short headline, brief subhead, 1–2 CTAs, and a product visual or screenshot. Mobile-first: stack content vertically, then use a two-column layout on larger screens.

<section class="hero" aria-labelledby="hero-title">
  <div class="hero-content">
    <h1 id="hero-title">Beautiful product. Simple pricing.</h1>
    <p class="muted">Ship faster with design-ready components and a simple API. Free 14-day trial — no credit card.</p>
    <div style="margin-top:12px">
      <a href="#get-started" class="btn">Get started free</a>
      <a href="#learn-more" class="btn ghost">Learn more</a>
    </div>
  </div>

  <div class="hero-visual">
    <img src="images/hero.png" alt="Product user interface screenshot">
  </div>
</section>

Design note: Put the hero visual after the content in the DOM for mobile so screen readers encounter the headline first; use CSS to re-order on wide screens if necessary.

8 — Core CSS (mobile-first)

Below is a compact but complete CSS sample covering layout, hero, features, pricing, and basic responsive rules. Put this in css/style.css.

/* css/style.css (core) */
:root{--primary:#007bff;--accent:#ff7a59;--bg:#f7f9fb;--radius:10px}
*{box-sizing:border-box}html,body{margin:0;background:var(--bg);font-family:Inter,Segoe UI,Arial,sans-serif;color:#222}
a{color:inherit}
.site-nav{display:flex;justify-content:space-between;align-items:center;padding:12px 0}
.logo{font-weight:700;font-size:1.1rem}
.nav-actions{display:flex;gap:8px;align-items:center}

/* Hero */
.hero{display:flex;flex-direction:column;padding:18px;gap:12px;align-items:flex-start}
.hero-visual img{width:100%;height:auto;border-radius:8px}
.hero .btn{margin-right:8px}

/* Features */
.features{display:grid;grid-template-columns:1fr;gap:12px;margin-top:12px}
@media(min-width:640px){.features{grid-template-columns:repeat(2,1fr)}}
@media(min-width:980px){.features{grid-template-columns:repeat(3,1fr)}}
.feature-card img{width:60px;height:60px;object-fit:cover;margin:0 auto 10px;display:block}

/* Pricing */
.pricing{display:flex;flex-direction:column;gap:12px;margin-top:12px}
@media(min-width:700px){.pricing{flex-direction:row}}
.price-card{padding:16px;border-radius:8px;border:1px solid #eee;text-align:center}

/* Testimonials */
.grid-cards{display:grid;grid-template-columns:1fr;gap:12px;margin-top:12px}
@media(min-width:700px){.grid-cards{grid-template-columns:repeat(2,1fr)}}

/* Utility */
.btn{display:inline-block;padding:10px 14px;border-radius:8px;background:var(--primary);color:#fff;text-decoration:none;font-weight:600}
.btn.ghost{background:#fff;color:var(--primary);border:1px solid #e6eefc}
.muted{color:#6c757d}

/* Responsive layout: hero two-column */
@media(min-width:900px){
  .hero{flex-direction:row;align-items:center}
  .hero .hero-content{flex:1}
  .hero .hero-visual{width:420px}
}

Notes: Keep CSS small and modular. Use variables for colors and easy theme changes.

9 — Accessibility & SEO tips

  • Use semantic landmarks: <header>, <main>, <section>, <footer>.
  • Provide descriptive alt text for images and aria-label where necessary.
  • Use clear H1 for the hero and H2 for section headings — one H1 per page.
  • Include meta title and description tailored to the product and audience.

10 — Performance & production checklist

  1. Optimize hero and feature images: use WebP and multiple sizes via srcset.
  2. Preconnect to fonts and use display=swap to avoid FOIT.
  3. Minify CSS & JS and enable gzip/Brotli on server.
  4. Set caching headers for static assets and use a CDN for global users.
  5. Defer nonessential JS (analytics) so hero and CTAs load quickly.

11 — Optional enhancements

  • Signup modal for the CTA using accessible dialog patterns (role="dialog").
  • Animated micro-interactions on CTA hover (scale, shadow).
  • AB test headlines and CTAs — measure conversions.

13 — Deployment & launch checklist

  1. Replace placeholder images with optimized WebP/JPEGs and add srcset.
  2. Preconnect to external fonts and use display=swap.
  3. Minify assets and enable server compression (gzip/Brotli).
  4. Configure caching headers and deploy to a CDN-enabled host (Netlify, Vercel, GitHub Pages for static sites).
  5. Set up analytics & get consent if required by law.

14 — Challenge tasks (practice)

  1. Add a pricing toggle (monthly/yearly) with JS that smoothly transitions prices.
  2. Implement an accessible signup modal for the CTA with keyboard focus trap.
  3. Add microcopy under the CTA with trust signals (e.g., "No credit card required").
  4. Create A/B test variations of the hero headline and measure conversions.
