Multi-Page Business / Agency Website

Introduction

Welcome! In this tutorial you will learn how to build a **multi-page business/agency website** for a fictional agency named Ejistan. The tutorial is structured, step-by-step, and each section includes ready-to-copy code blocks. We will build four pages:

  1. index.html — Homepage
  2. about.html — About / Team
  3. services.html — Services detail
  4. contact.html — Contact form & capture

Classroom style: each section explains reasoning, accessibility and deployment tips. Copy code with the "Copy" buttons and test in Acode or your editor.

Project Plan & Folder Structure

Planning first avoids messy projects later. Create a folder named ejistan and inside it create this structure: 

/ejistan/
  index.html
  about.html
  services.html
  contact.html
  css/
    style.css
  js/
    main.js
  images/
    logo.png
    hero.jpg
    team-1.jpg
    work-case1.jpg
  docs/
    privacy.pdf

Why this structure? - CSS, JS and images are clearly separated. It makes deployment simple (upload the root folder). Use small, optimized images and name files with hyphens.

Common HTML Head & Shared Elements

Every page will include a consistent <head> block. Copy this into every page's head to keep consistency, fonts, and meta tags. 

<!-- common-head.html (paste into each page's head) -->
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>Ejistan — Digital Agency</title>
<meta name="description" content="Ejistan is a full-service digital agency offering product design, web development and growth services.">

<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;600;700&display=swap" rel="stylesheet">

<script defer src="js/main.js"></script>

Note: Using a common-head snippet helps keep metadata consistent and improves SEO management across pages.

Navigation & Site Header (shared partial)

The header/navigation is consistent across pages. Use a semantic <header> with a nav and accessible attributes. 

<!-- header partial (include on every page) -->
<header class="site-header">
  <div class="container">
    <a href="index.html" class="brand"><img src="images/logo.png" alt="Ejistan logo" height="36"></a>

    <nav aria-label="Main navigation">
      <ul class="nav-links">
        <li><a href="index.html">Home</a></li>
        <li><a href="about.html">About</a></li>
        <li><a href="services.html">Services</a></li>
        <li><a href="work.html">Work</a></li>
        <li><a href="contact.html">Contact</a></li>
      </ul>
    </nav>

    <a href="contact.html" class="btn btn-primary">Request Quote</a>
  </div>
</header>

Accessibility note: Keep markup order logical: brand < nav > CTA. Use aria-label to name the navigation region for assistive technology.

Client-side JS (optional) — js/main.js

Keep JavaScript minimal. This small file handles form UX (preventing double-submit) and adds a mobile nav toggle if you later add a hamburger menu. 

// js/main.js (minimal)
document.addEventListener('DOMContentLoaded', function() {
  // Prevent double form submit
  const forms = document.querySelectorAll('form');
  forms.forEach(form => {
    form.addEventListener('submit', function(e) {
      const btn = form.querySelector('button[type="submit"]');
      if (btn) {
        btn.disabled = true;
        btn.textContent = 'Sending...';
      }
    });
  });

  // Mobile nav toggle (optional)
  const navToggle = document.querySelector('.nav-toggle');
  if (navToggle) {
    navToggle.addEventListener('click', () => {
      const nav = document.querySelector('.nav-links');
      nav.classList.toggle('open');
    });
  }
});

Note: This JS is intentionally tiny — avoid loading heavy libraries unless needed. Defer or load scripts at the end of the page to avoid blocking rendering.

Accessibility & SEO Checklist (Classroom Notes)

Before you ship Ejistan, run this checklist. These are core, non-negotiable items for a professional site.

  • Unique <title> & meta description per page
  • One logical <h1> per page; use H2/H3 for subheadings
  • Alt attributes for all meaningful images; decorative images use empty alt alt=""
  • Accessible forms (labels, required attributes, error messages)
  • Keyboard focus styles visible and usable
  • ARIA only when necessary — prefer native HTML
  • Structured data (JSON-LD) for Organization and CaseStudy where appropriate
