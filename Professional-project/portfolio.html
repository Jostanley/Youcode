Building a Portfolio Website

1 — Goal of the Project

You will create your **first professional website** — a clean, responsive **portfolio** to showcase your name, photo, skills, projects, and contact form. It combines everything you’ve learned: semantic HTML, CSS grid/flexbox, responsive design, accessibility, and performance optimization.

Outcome: At the end you’ll have an index.html and style.css ready to upload to GitHub Pages or Render static hosting.

2 — Plan and Folder Setup

Create a new folder on your device (for example portfolio-site) and inside it:

  • index.html – main file
  • style.css – external CSS
  • images/ – profile photo & project screenshots

Link your CSS file inside <head>:

<link rel="stylesheet" href="style.css">

3 — HTML Structure (Semantic Layout)

We’ll divide the page into clear landmarks — <header>, <nav>, <main> with sections for Hero, About, Projects, Contact, and <footer>.

4 — Styling Your Portfolio (style.css)

Now design it with modern CSS — colors, fonts, layout, and responsive grid.

/* Global Reset */

5 — Accessibility and Best Practices

  • Add alt text to all images.
  • Ensure keyboard navigation works for links and buttons.
  • Maintain color contrast (blue on white passes WCAG AA).
  • Use meaningful headings for screen reader hierarchy.

6 — Mobile Optimization

Already mobile-first, but test on Android and small screens.

  • Images resize with width:100%.
  • Buttons large enough for touch (≥44 px height).
  • Meta viewport tag set for scaling.

7 — Personalization Ideas

Add these to make it yours:

  • Replace blue with your brand color.
  • Use Google Fonts (Poppins, Inter).
  • Add a “Download CV” button linking to PDF.
  • Include social icons (GitHub, LinkedIn, X/Twitter).

8 — Deployment

  1. Create a GitHub repository named portfolio.
  2. Upload index.html, style.css, and images.
  3. Enable GitHub Pages under Settings → Pages → Deploy from branch.
  4. Share your live URL (e.g. https://username.github.io/portfolio).

Alternative hosting: Render, Netlify, or your own domain (e.g. kesug.com).

9 — Optional Enhancements

  • Dark Mode toggle using CSS variables and JS.
  • Scroll Animations via AOS library or simple fade-in CSS.
  • Form Handling with Formspree or Netlify Forms (no backend needed).
  • Accessibility Testing with Lighthouse/Axe extension.

10 — Challenge Tasks for Students

  1. Add a third project card with unique content and image.
  2. Implement a sticky navbar that shrinks on scroll.
  3. Create a “Back to Top” button visible after scroll 500 px.
  4. Validate your HTML and CSS with the W3C validators.
  5. Compress images for faster loading (less than 200 KB each).

11 — Conclusion

Congratulations! You’ve built a professional portfolio that works on mobile, tablet, and desktop. This is your first real project to show potential clients or employers. Keep updating it as you learn new skills and complete more projects.
